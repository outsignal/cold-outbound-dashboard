    **File 1: `src/app/api/analytics/benchmarks/icp-calibration/route.ts`**

    Create GET endpoint accepting optional `workspace` query param and optional `global` boolean param:

    1. `await requireAdminAuth()` at top
    2. Use raw SQL (`prisma.$queryRaw`) to cross-join PersonWorkspace (icpScore) with Reply outcomes:
       - JOIN path: LeadWorkspace -> Lead (by leadId) -> Reply (by Lead.email = Reply.senderEmail AND LeadWorkspace.workspace = Reply.workspaceSlug)
       - Group into 5 buckets: 0-20, 21-40, 41-60, 61-80, 81-100
       - Count: total people per bucket, reply count, interested count (intent IN ('interested', 'meeting_booked'))
       - Filter by workspace if provided (unless global=true)
    3. Compute rates per bucket: replyRate = (replyCount / totalPeople) * 100, interestedRate = (interestedCount / totalPeople) * 100
    4. Compute threshold recommendation using this logic:
       - Need at least 50 people with ICP scores total to make any recommendation (return null if < 50)
       - Find the current workspace threshold: query Campaign.icpScoreThreshold for the workspace's signal campaigns. Use most common value, or default 70 if no signal campaigns
       - Walk buckets from highest to lowest. Find where interestedRate drops below 50% of the peak bucket's interestedRate. The bottom of that last "good" bucket becomes the recommended threshold
       - Confidence: high (200+ people), medium (100-199), low (50-99)
       - Build evidence string with specific numbers: "Data suggests raising threshold from 60 to 72. People scoring 61+ have 4.2% interested rate vs 1.1% for lower buckets (based on 156 scored leads)."
    5. Return structured empty response if < 50 ICP-scored people: `{ buckets: [], recommendation: null, totalPeople: N, message: "Not enough ICP data (need 50+, currently N)" }`

    Response shape:
    ```typescript
    {
      buckets: Array<{ bucket: string; totalPeople: number; replyRate: number; interestedRate: number }>;
      recommendation: { currentThreshold: number; recommendedThreshold: number; evidence: string; confidence: "high" | "medium" | "low"; sampleSize: number } | null;
      totalPeople: number;
      workspace: string | null;
      isGlobal: boolean;
    }
    ```

    Create GET endpoint accepting optional `workspace` query param and optional `global` boolean param:

    1. `await requireAdminAuth()` at top
    2. Query all campaigns where `type = 'signal'`, filtered by workspace if provided (unless global=true)
    3. For each signal campaign, parse `signalTypes` JSON string (try/catch, default []) to get the signal types array
    4. Query CachedMetrics campaign_snapshot for each signal campaign to get sent/replied/interested counts
    5. Also query Reply table for intent classification counts per signal campaign
    6. Aggregate per signal type: total sent, total replied, total interested, reply rate, interested rate. Only include signal types with 5+ leads (hide < 5). Add `lowConfidence: true` badge for 5-9 leads
    7. Rank signal types by interested rate descending
    8. Compute static campaign baseline: query non-signal campaigns' aggregate rates for comparison
    9. Build signal vs static comparison: { signalAvgReplyRate, signalAvgInterestedRate, staticAvgReplyRate, staticAvgInterestedRate, signalMultiplier (signal/static ratio) }
    10. Return structured empty response if 0 signal campaigns: `{ signalTypes: [], comparison: null, message: "No signal campaigns configured" }`
    11. Return structured empty response for comparison if 0 static campaigns with data: comparison field null with message

    Response shape:
    ```typescript

      signalTypes: Array<{
        type: string; // "funding", "hiring_spike", etc.
        sent: number; replied: number; interested: number;
        replyRate: number; interestedRate: number;
        lowConfidence: boolean;
      }>;
      comparison: {
        signalAvg: { replyRate: number; interestedRate: number; campaigns: number };
        staticAvg: { replyRate: number; interestedRate: number; campaigns: number };
        multiplier: { replyRate: number; interestedRate: number };
      } | null;
      workspace: string | null;
      isGlobal: boolean;
    }
    ```
