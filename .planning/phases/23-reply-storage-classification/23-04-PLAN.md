All components are client components ("use client").

**intent-badge.tsx:**
- Renders a color-coded pill showing the effective intent label (using INTENT_LABELS and INTENT_COLORS from types.ts)
- Two modes: `display` (just shows the badge) and `editable` (click opens a dropdown to override)
- In editable mode, clicking the badge shows a dropdown with all 9 intent options. Selecting one calls `onOverride(newIntent)` callback.
- If the reply has been overridden, show a small "edited" indicator (tiny pencil icon from lucide-react)
- Props: `intent: string | null`, `overrideIntent: string | null`, `editable?: boolean`, `onOverride?: (intent: string) => void`

**sentiment-badge.tsx:**
- Simpler pill: positive (green dot + "Positive"), neutral (gray dot + "Neutral"), negative (red dot + "Negative")
- Uses SENTIMENT_COLORS from types.ts
- Props: `sentiment: string | null`, `overrideSentiment: string | null`

**reply-table.tsx:**
- Dense table layout. Columns: Sender (name + email), Subject, Intent (IntentBadge), Sentiment (SentimentBadge), Workspace (small tag), Time (relative like "2h ago"), Preview (first ~100 chars of bodyText, truncated)
- Each row is clickable -- calls `onSelect(reply)` callback
- Selected row gets a subtle highlight (bg-muted)
- Show a loading skeleton when `loading` prop is true
- Props: `replies: Reply[]`, `onSelect: (reply: Reply) => void`, `selectedId: string | null`, `loading: boolean`
- Use `Intl.RelativeTimeFormat` or simple relative time helper for timestamps

**reply-side-panel.tsx:**
- Slide-out panel from the right (use a div with `translate-x` transition, not a dialog/sheet -- simpler and matches the dense layout). When a reply is selected, panel slides in from right. When deselected, slides out.
- Sections:
  1. **Header**: Sender name + email, timestamp (absolute), source badge (webhook/poll)
  2. **Classification**: IntentBadge (editable mode), SentimentBadge, objection subtype if applicable, classification summary text
  3. **Reply Content**: Full bodyText displayed in a pre-wrap div with monospace-ish font
  4. **Original Outbound**: If outboundSubject/outboundBody available, show in a collapsible section with lighter background
  5. **Campaign**: Campaign name (if available), sequence step number
  6. **Links**: "Reply in Outsignal" button linking to `https://app.outsignal.ai/inbox` (opens new tab)
- Override flow: When admin clicks intent badge dropdown and selects a new intent, call PATCH /api/replies/:id with the override. On success, update the local state. Show a small toast or inline confirmation.
- Close button (X) in top-right corner
- Props: `reply: Reply | null`, `onClose: () => void`, `onOverrideSuccess: (updatedReply: Reply) => void`

**reply-stats.tsx:**
- Renders at the top of the replies page
- Three sections in a row (grid cols-3 on lg, stack on mobile):
  1. **Total count card**: Total replies, classified count, unclassified count
  2. **Intent distribution chart**: Horizontal bar chart (recharts BarChart) or pie chart showing intent category counts. Use INTENT_COLORS for bar fills.
  3. **Sentiment bar**: Simple stacked horizontal bar showing positive/neutral/negative proportions
- Props: `stats: StatsResponse | null`, `loading: boolean`
- Keep charts compact (max height ~150px) since this is a summary strip, not the main content

Follow the webhook-log page pattern. "use client" with nuqs for URL state.

**URL state (nuqs):**
- `workspace`: parseAsString (workspace filter)
- `intent`: parseAsString (comma-separated intent filter)
- `sentiment`: parseAsString (sentiment filter)
- `search`: parseAsString (search query)
- `range`: parseAsString, default "all" (date range)
- `page`: parseAsInteger, default 1

**Layout:**
1. Page header: "Replies" title with total count
2. Filter bar:
   - Workspace dropdown (fetch workspace list from existing endpoint or hardcode the 7 known workspaces)
   - Intent filter chips using ToggleChip pattern (one chip per intent category, multi-select). Active chips add to the comma-separated `intent` URL param.
   - Sentiment filter: 3 toggle buttons (Positive, Neutral, Negative)
   - Date range: preset buttons (24h, 7d, 30d, All)
   - Search input with debounce (use-debounce, 300ms)
3. Stats strip: ReplyStats component
4. Main content: ReplyTable component
5. Pagination: simple prev/next with page number display
6. Side panel: ReplySidePanel (fixed position, slides from right)

**Data fetching:**
- Fetch replies from `/api/replies` with all filter params as query string
- Fetch stats from `/api/replies/stats` with workspace, campaignId, and range params
- Use `useEffect` triggered by filter changes (debounced search)
- Show loading skeleton during fetch

**Override flow:**
- When admin overrides a classification in the side panel, the component calls PATCH /api/replies/:id
- On success, update the reply in the local replies array and refetch stats (since distributions changed)

**Page metadata:**
- Add to the admin sidebar navigation if there's a nav config file. Otherwise just ensure the page is accessible at /admin/replies.
