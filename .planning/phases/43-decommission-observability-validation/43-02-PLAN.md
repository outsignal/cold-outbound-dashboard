**Webhook handler cleanup (src/app/api/webhooks/emailbison/route.ts):**

1. Find the `notify({ type: "system" }).catch(() => {})` calls for BOUNCE and UNSUBSCRIBED events (around lines 438 and 454 per research).
2. Convert each from:
   ```
   notify({ ... }).catch(() => {});
   ```
   to:
   ```
   try {
     await notify({ ... });
   } catch (err) {
     console.error("[webhook] System notification failed:", err);
   }
   ```
   This makes the notification awaited (not fire-and-forget) but still non-throwing (handler continues on notification failure).

3. Do a full scan of the file for any other `.then(` or `.catch(() => {})` patterns that indicate fire-and-forget. Convert them similarly.
