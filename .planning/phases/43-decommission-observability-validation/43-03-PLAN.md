Create `src/app/api/background-tasks/route.ts` with a GET handler:

1. Import `requireAdminAuth` from `@/lib/require-admin-auth`. Call it at the start — return 401 if unauthorized.

2. Read query params:
   - `period` (default: `"1d"`) — maps to Trigger.dev `filter[createdAt][period]`
   - `workspace` (optional) — maps to `filter[tag]`

3. Create a helper function `triggerFetch(path: string)`:
   ```
   const TRIGGER_API_BASE = "https://api.trigger.dev/api/v1";
   async function triggerFetch(path: string) {
     const res = await fetch(`${TRIGGER_API_BASE}${path}`, {
       headers: { Authorization: `Bearer ${process.env.TRIGGER_SECRET_KEY}` },
       next: { revalidate: 0 },
     });
     if (!res.ok) throw new Error(`Trigger.dev API ${res.status}`);
     return res.json();
   }
   ```

4. Fetch runs and schedules in parallel via `Promise.all`:
   - Runs: `GET /runs?filter[createdAt][period]=${period}&page[size]=100` (add `filter[tag]=${workspace}` if workspace is set)
   - Schedules: `GET /schedules`

5. Compute summary from runs:
   - `total`: total run count
   - `succeeded`: runs with status `COMPLETED`
   - `failed`: runs with status `FAILED`, `CRASHED`, or `SYSTEM_FAILURE`
   - `running`: runs with status `EXECUTING`, `REATTEMPTING`, or `QUEUED`
   - `activeSchedules`: schedules where `active === true`

6. Return JSON:
   ```
   {
     summary: { total, succeeded, failed, running, activeSchedules },
     runs: runs.slice(0, 50),  // cap for frontend performance
     schedules: schedulesData.data.filter(s => s.active),
   }
   ```

7. Wrap the entire handler in try/catch — return 500 with error message on failure. Log the error with `console.error("[background-tasks]", err)`.
