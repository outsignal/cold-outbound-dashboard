**In `src/lib/linkedin/sequencing.ts` — fix `createSequenceRulesForCampaign`:**

Change line 205 from:
```typescript
triggerStepRef: step.triggerStepRef ?? null,
```
to:
```typescript
triggerStepRef: step.triggerEvent === "email_sent" || (!step.triggerEvent && step.position !== 1)
  ? (step.triggerStepRef ?? `email_${step.position}`)
  : (step.triggerStepRef ?? null),
```

This derives `triggerStepRef` from the step position for email_sent-triggered rules. The logic:
- If `triggerEvent` is explicitly `"email_sent"` → derive `email_{position}`
- If `triggerEvent` is NOT set AND position is NOT 1 (position 1 defaults to `delay_after_previous`) → derive `email_{position}`
- Otherwise, keep any explicit `triggerStepRef` or null

This ensures the webhook's `triggerStepRef = "email_1"` query will match rules created during deploy.
