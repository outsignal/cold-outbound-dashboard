**Exports:**

1. `PromotionResult` interface:
```typescript
export interface PromotionResult {
  promoted: number;
  duplicates: number;
  duplicateNames: string[];   // Up to 5 sample names for display
  promotedIds: string[];      // Person IDs of promoted leads
  enrichmentJobId?: string;   // EnrichmentJob ID if enrichment was enqueued
}
```

2. `deduplicateAndPromote(workspaceSlug: string, runIds: string[]): Promise<PromotionResult>` â€” the main function:
   - Fetch all `DiscoveredPerson` records with `status: "staged"` and `discoveryRunId in runIds`
   - For each staged record, call `findExistingPerson(dp)` to check for duplicates
   - If duplicate found: update DiscoveredPerson `status` to `"duplicate"`, set `personId` to the matching Person ID, set `promotedAt` to `new Date()`
   - If not duplicate: call `promoteToPerson(dp, workspaceSlug)` to create Person + PersonWorkspace, update DiscoveredPerson `status` to `"promoted"`, set `personId`, set `promotedAt`
   - After all records processed: if any promoted, call `triggerEnrichmentForPeople(promotedIds, workspaceSlug)`
   - Return the PromotionResult

3. `findExistingPerson(dp)` â€” internal function, three-leg match:
   - **Leg 1: Email exact match** â€” if `dp.email` is not null and not a placeholder (`!dp.email.includes('@discovery.internal')`), do `prisma.person.findUnique({ where: { email: dp.email } })`. If found, return `person.id`.
   - **Leg 2: LinkedIn URL exact match** â€” if `dp.linkedinUrl` is not null, do `prisma.person.findFirst({ where: { linkedinUrl: dp.linkedinUrl } })`. If found, return `person.id`.
   - **Leg 3: Fuzzy name+company match** â€” only when ALL three are present: `dp.firstName`, `dp.lastName`, AND `dp.companyDomain`. Query `prisma.person.findMany({ where: { companyDomain: dp.companyDomain }, select: { id: true, firstName: true, lastName: true }, take: 100 })`. For each candidate, compute string similarity of full names using `stringSimilarity()`. If any candidate has similarity >= 0.85, return its ID.
   - Return `null` if no leg matches.

4. `promoteToPerson(dp, workspaceSlug)` â€” internal function:
   - Generate email: use `dp.email` if present, otherwise `placeholder-${randomUUID()}@discovery.internal` (import `randomUUID` from `"crypto"`)
   - Upsert Person: `prisma.person.upsert({ where: { email }, create: { email, firstName, lastName, jobTitle, company, companyDomain, linkedinUrl, phone, location, source: "discovery-" + dp.discoverySource, status: "new" }, update: {} })`
   - Upsert PersonWorkspace: `prisma.personWorkspace.upsert({ where: { personId_workspace: { personId: person.id, workspace: workspaceSlug } }, create: { personId: person.id, workspace: workspaceSlug }, update: {} })`
   - Return the person record (needs at minimum `{ id: string }`)

5. `triggerEnrichmentForPeople(personIds, workspaceSlug)` â€” internal function:
   - If personIds is empty, return early
   - Call `enqueueJob({ entityType: "person", provider: "waterfall", entityIds: personIds, chunkSize: 25, workspaceSlug })`
   - Return the job ID

6. Hand-rolled `levenshteinDistance(a: string, b: string): number` â€” standard dynamic programming implementation (~25 lines). No external dependency.

7. `stringSimilarity(a: string, b: string): number` â€” `1 - levenshteinDistance(a, b) / Math.max(a.length, b.length)`. Returns 1.0 for identical strings, 0.0 for completely different.
