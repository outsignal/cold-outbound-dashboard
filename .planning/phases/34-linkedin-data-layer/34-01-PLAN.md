```prisma
// --- LinkedIn Inbox Data Layer (v5.0 Phase 34) ---

model LinkedInConversation {
  id                     String   @id @default(cuid())
  conversationId         String   @unique  // LinkedIn's conversation ID (from entityUrn)
  entityUrn              String            // Full LinkedIn entityUrn
  senderId               String            // Which sender account fetched this
  workspaceSlug          String            // Denormalized for workspace-scoped queries
  personId               String?           // Matched Person record (null if no URL match)

  // Participant metadata
  participantName        String?
  participantUrn         String?
  participantProfileUrl  String?           // Normalized to /in/username format
  participantHeadline    String?
  participantProfilePicUrl String?

  // Message preview
  lastMessageSnippet     String?
  lastActivityAt         DateTime
  unreadCount            Int      @default(0)

  createdAt              DateTime @default(now())
  updatedAt              DateTime @updatedAt

  sender    Sender             @relation(fields: [senderId], references: [id], onDelete: Cascade)
  messages  LinkedInMessage[]

  @@index([workspaceSlug, lastActivityAt])
  @@index([senderId])
  @@index([personId])
}

model LinkedInMessage {
  id               String   @id @default(cuid())
  conversationId   String            // FK to LinkedInConversation.id (internal cuid)
  eventUrn         String   @unique  // LinkedIn's eventUrn (dedup key)
  senderUrn        String            // LinkedIn URN of message sender
  senderName       String?
  body             String
  isOutbound       Boolean           // true = we sent it, false = they sent it
  deliveredAt      DateTime

  createdAt        DateTime @default(now())

  conversation LinkedInConversation @relation(fields: [conversationId], references: [id], onDelete: Cascade)

  @@index([conversationId, deliveredAt])
}

model LinkedInSyncStatus {
  id                String    @id @default(cuid())
  senderId          String    @unique
  lastSyncedAt      DateTime?
  conversationCount Int       @default(0)
  createdAt         DateTime  @default(now())
  updatedAt         DateTime  @updatedAt

  sender Sender @relation(fields: [senderId], references: [id], onDelete: Cascade)
}
```

**1. Create src/lib/linkedin/sync.ts** with the core sync function:

The function `syncLinkedInConversations(senderId: string): Promise<void>` must:

a. Call the Railway worker: `GET ${WORKER_URL}/sessions/${senderId}/conversations` with Bearer auth using `WORKER_API_SECRET`. If the worker returns non-200, log error and return (do not throw — fire-and-forget must not crash).

b. Parse response as `{ conversations: VoyagerConversation[] }`. If parsing fails or conversations is empty, return gracefully.

c. Look up the sender to get `workspaceSlug`: `prisma.sender.findUnique({ where: { id: senderId }, select: { workspaceSlug: true } })`. If sender not found, return.

d. For each conversation, do Person matching:
   - Use `normalizeLinkedinUrl(conv.participantProfileUrl)` to extract `/in/username` (lowercase, no trailing slash)
   - Query: `prisma.person.findFirst({ where: { linkedinUrl: { contains: normalizedUrl } }, select: { id: true } })`
   - If no match, `personId = null` (store conversation anyway per CONTEXT.md decision)

e. Upsert each conversation using `prisma.linkedInConversation.upsert()`:
   - `where: { conversationId: conv.conversationId }`
   - `create`: all fields including personId
   - `update`: lastActivityAt, unreadCount, lastMessageSnippet, participantName, participantHeadline, participantProfilePicUrl (do NOT update personId — initial match is authoritative per CONTEXT.md)
   - Store `participantProfileUrl` as the normalized form

f. After upserting all conversations, update sync status:
   ```typescript
   await prisma.linkedInSyncStatus.upsert({
     where: { senderId },
     create: { senderId, lastSyncedAt: new Date(), conversationCount: syncCount },
     update: { lastSyncedAt: new Date(), conversationCount: syncCount },
   });
   ```

g. Wrap the entire function body in try/catch. On error, `console.error('[linkedin-sync] Failed for sender ${senderId}:', err)` and return (no rethrow).

Helper function `normalizeLinkedinUrl(url: string | null): string | null`:
- Return null if input is null/empty
- Match `/in/([^/?#]+)` from the URL
- Return `/in/${match[1].toLowerCase()}` (lowercase, no trailing slash)
- Return null if no match found

Import types from `@/lib/linkedin/types` (VoyagerConversation), NOT from worker/.

**2. Create src/app/api/portal/inbox/linkedin/sync/route.ts** with POST handler:

The route must:

a. Authenticate: `const { workspaceSlug } = await getPortalSession()`. On error, return 401.

b. Load LinkedIn senders: `prisma.sender.findMany({ where: { workspaceSlug, status: "active" }, select: { id: true } })`. Note: do NOT filter by `sessionStatus: "active"` — senders with expired sessions should still show their previously synced conversations. The sync function handles worker errors gracefully.

c. Load sync statuses: `prisma.linkedInSyncStatus.findMany({ where: { senderId: { in: senders.map(s => s.id) } } })`

d. Compute which senders need sync (5-min cooldown):
   ```typescript
   const COOLDOWN_MS = 5 * 60 * 1000;
   const now = Date.now();
   const sendersToSync = senders.filter(s => {
     const status = syncStatuses.find(ss => ss.senderId === s.id);
     if (!status?.lastSyncedAt) return true;
     return now - status.lastSyncedAt.getTime() > COOLDOWN_MS;
   });
   ```

e. Fetch existing conversations from DB (always return these):
   ```typescript
   const conversations = await prisma.linkedInConversation.findMany({
     where: { workspaceSlug },
     orderBy: { lastActivityAt: "desc" },
     take: 50,
   });
   ```

f. Compute lastSyncedAt as the most recent timestamp across all sender sync statuses.

g. If sendersToSync is not empty, fire sync: `void Promise.allSettled(sendersToSync.map(s => syncLinkedInConversations(s.id)))`. This starts the worker calls but does NOT await them.

h. Return response:
   - Status 202 if any senders are syncing, 200 if all within cooldown
   - Body: `{ conversations, lastSyncedAt, syncing: boolean }`

Per CONTEXT.md: "return 200 with existing DB conversations (not a rejection)" when all within cooldown. When syncing, return 202 with existing conversations (client shows existing data instantly, polling picks up fresh data).
