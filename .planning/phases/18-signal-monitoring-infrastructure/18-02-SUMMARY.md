- predictLeadsGet() retries on 429 and AbortError (timeout) only — other HTTP errors (401, 403, 404, 500) thrown immediately as non-retryable application errors
- PredictLeadsListResponseSchema implemented as generic factory function rather than pre-instantiated schema — enables typed inference for each signal type
- Dockerfile copies prisma/ from repo root — required for prisma generate since Prisma schema lives in the main project; Railway build context must be repo root
