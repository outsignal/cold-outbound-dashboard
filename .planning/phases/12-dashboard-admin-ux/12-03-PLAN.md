Create `src/app/api/senders/route.ts`:
- **GET**: Returns all senders across all workspaces with workspace name included. Accepts optional `workspace` query param to filter by slug. Includes related workspace name via Prisma include. Orders by workspaceSlug, then name.
  ```typescript
  // Response shape:
  { senders: Array<Sender & { workspace: { name: string } }> }
  ```
- **POST**: Creates a new sender. Required fields: `workspaceSlug`, `name`. Optional fields: `emailAddress`, `linkedinProfileUrl`, `linkedinEmail`, `proxyUrl`, `linkedinTier`, `dailyConnectionLimit`, `dailyMessageLimit`, `dailyProfileViewLimit`. Validates workspaceSlug exists. Returns 201 with created sender.

Create `src/app/api/senders/[id]/route.ts`:
- **GET**: Returns single sender by ID with workspace info.
- **PATCH**: Updates sender fields. Accepts any subset of: `name`, `emailAddress`, `linkedinProfileUrl`, `linkedinEmail`, `proxyUrl`, `linkedinTier`, `dailyConnectionLimit`, `dailyMessageLimit`, `dailyProfileViewLimit`, `status` (for pause/unpause: accepts "active" or "paused"). Validates status values against allowed set.
- **DELETE**: Deletes sender by ID. Only allowed if sender has no pending/running LinkedInActions (check first, return 409 if blocked). Returns 204 on success.

Both routes use `import { prisma } from "@/lib/db"`.
