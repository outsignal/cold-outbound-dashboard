  <action>
Create GET `/api/dashboard/stats` endpoint that accepts query params:
- `workspace` (optional) — filter by workspace slug, default "all"
- `days` (optional) — time range in days, default 7. Support values: 7, 14, 30, 90.

The endpoint queries:
1. **Email KPIs** from WebhookEvent: count by eventType (EMAIL_SENT, EMAIL_OPENED, LEAD_REPLIED, LEAD_INTERESTED, BOUNCED) within time range, filtered by workspace if specified.
2. **LinkedIn KPIs** from LinkedInAction: count by actionType (connect, message, profile_view) and status (complete, pending, failed) within time range, filtered by workspace.
3. **Pipeline KPIs** from PersonWorkspace: count by status (contacted, replied, interested) filtered by workspace.
4. **Health KPIs**: count of Sender records by healthStatus and sessionStatus, filtered by workspace.
5. **Inbox connected/disconnected**: count of workspaces with/without apiToken.
6. **Time-series data**: group WebhookEvent by date within range, broken down by eventType — for the line chart. Return array of `{ date: string, sent: number, replies: number, bounces: number, opens: number }`.
7. **Alerts**: query Sender where healthStatus != 'healthy', AgentRun where status='failed' and createdAt > 24h ago, Workspace where apiToken is null but status='active'.

Return JSON: `{ kpis: {...}, timeSeries: [...], alerts: [...], workspaces: [{slug, name}] }`.

- Renders a Select dropdown with "All Campaigns" as default, plus one option per workspace (slug + name).
- Uses `nuqs` (already in package.json) with `useQueryState` for the `workspace` and `days` URL params.
- Also renders a time range Select: "Last 7 days", "Last 14 days", "Last 30 days", "Last 90 days".
- Both dropdowns trigger page re-render via URL param change.
- Compact layout: both selects inline, right-aligned.
