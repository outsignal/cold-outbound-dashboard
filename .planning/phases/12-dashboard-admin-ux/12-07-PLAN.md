Instead, accept the uploaded file content as text/plain or have the client-side convert to text before uploading (using browser-based PDF.js). OR: just accept any text content (paste from Google Doc, raw text from PDF viewer) and parse it.
- **Revised approach**: The upload endpoint accepts JSON `{ content: string, fileName: string, type: "proposal" | "onboarding" }`. The content is pre-extracted text (user pastes from PDF viewer or Google Doc). The endpoint uses a simple heuristic parser to extract fields:
  - Look for patterns: "Client Name:", "Company:", "Email:", "Package:", etc.
  - Return parsed fields as suggestions: `{ parsed: { clientName?, clientEmail?, companyOverview?, packageType?, ... }, raw: string }`.
  - The client-side form pre-fills with parsed fields, user reviews and edits before saving.
- This avoids PDF parsing dependencies entirely — the smart parse happens on extracted text.
