Create GET `/api/people/[id]/timeline` endpoint:
- Accepts person ID as route param.
- Queries in parallel:
  1. WebhookEvent where leadEmail = person.email, ordered by receivedAt desc, limit 100
  2. LinkedInAction where personId = person.id, ordered by createdAt desc, limit 100
  3. EnrichmentLog where entityId = person.id and entityType = "person", ordered by runAt desc, limit 50
- Normalizes all into a unified timeline format:
  ```typescript
  interface TimelineEvent {
    id: string;
    type: 'email_sent' | 'email_opened' | 'email_replied' | 'email_bounced' | 'linkedin_connect' | 'linkedin_message' | 'linkedin_profile_view' | 'enrichment' | 'other';
    title: string;
    detail?: string;
    workspace?: string;
    timestamp: string; // ISO
    metadata?: Record<string, unknown>;
  }
  ```
- Maps WebhookEvent.eventType to timeline types: EMAIL_SENT -> email_sent, EMAIL_OPENED -> email_opened, LEAD_REPLIED -> email_replied, BOUNCED -> email_bounced, etc.
- Maps LinkedInAction.actionType to timeline types: connect -> linkedin_connect, message -> linkedin_message, profile_view -> linkedin_profile_view.
- Maps EnrichmentLog to type 'enrichment' with provider as title detail.
- Merges all events, sorts by timestamp desc.
- Returns JSON: `{ events: TimelineEvent[], total: number }`.
