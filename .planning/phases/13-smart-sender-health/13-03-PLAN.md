Implementation:

- Fetch sender by ID. Return 404 if not found.
- Check `sender.healthStatus` is `"blocked"` or `"session_expired"`. If not, return 400: "Sender is not in a hard-flagged state".

  - Create `SenderHealthEvent` with status "healthy", reason "admin_reactivated", detail "Manually reactivated by admin"

- Wrap in try/catch, return 500 on error.

**2. GET /api/senders/[id]/health-history**
File: `src/app/api/senders/[id]/health-history/route.ts`

```typescript
import { NextRequest, NextResponse } from "next/server";
import { prisma } from "@/lib/db";
```

Implementation:
- Extract `id` from params.
- Query `SenderHealthEvent` for this sender, last 30 days, ordered by `createdAt desc`, limit 100.
- Also compute sparkline data: aggregate events by day for the last 30 days. For each day, take the worst status seen (healthy=0, warning=1, blocked/session_expired=2). Fill missing days with 0 (healthy). Result: 30 data points as `Array<{ date: string; statusNum: number }>`.
- Return `{ events: [...], sparkline: [...], summary: { currentStatus, lastFlagReason, flagCount, daysSinceLastIncident } }`.
  - `lastFlagReason`: reason from most recent non-healthy event (or null if never flagged).
  - `flagCount`: total non-healthy events in 30 days.
  - `daysSinceLastIncident`: days since last non-healthy event (or null if never flagged).
- Wrap in try/catch, return 500 on error.

Both endpoints follow the existing pattern in `src/app/api/senders/[id]/route.ts`.
