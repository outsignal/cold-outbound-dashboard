2. `checkBlacklists(domain: string, ip?: string): Promise<BlacklistResult>`

`BlacklistResult`: `{ domain: string; hits: Array<{ list: string; tier: "critical" | "warning"; delistUrl?: string }>; checkedAt: Date }`

For domain-based DNSBLs (like dbl.spamhaus.org):
- Reverse domain not needed — query `{domain}.{dnsbl}` for domain blacklists
- Actually: URI DNSBLs use the domain directly, IP DNSBLs use reversed IP

For IP-based DNSBLs:
- Reverse the IP octets: 1.2.3.4 → 4.3.2.1
- Query `{reversed_ip}.{dnsbl}`
- IP comes from `EMAILBISON_SENDING_IP` env var

For each DNSBL, do a DNS A record lookup. If it resolves (any A record), the domain/IP is listed. If NXDOMAIN or timeout, it's clean.

Use `dns.promises.Resolver` with 3s timeout. Run all 20 checks in parallel via `Promise.allSettled`. Log individual failures but don't fail overall.

3. `reverseIp(ip: string): string` — utility to reverse IP octets.

4. Separate domain-only DNSBLs (dbl.spamhaus.org) from IP-only DNSBLs (most others). Some check both — check domain against domain DNSBLs and IP against IP DNSBLs.

export async function GET(request: Request) {
  // 1. Validate cron secret
  // 2. Collect all unique sending domains:
  //    - Query Sender model where emailAddress is not null
  //    - Extract domains from email addresses
  //    - Deduplicate
  // 3. For each domain (up to 4 per run per CONTEXT.md progressive checking):
  //    - Prioritize: domains with bounce issues (check BounceSnapshot for high bounce rates)
  //      OR domains with oldest lastDnsCheck (or never checked)
  //    - Run checkAllDns(domain)
  //    - Run checkBlacklists(domain, process.env.EMAILBISON_SENDING_IP) — targeted per CONTEXT.md:
  //      only for domains with >3% bounce rate or not checked in 7+ days
  //    - Compute overallHealth
  //    - Upsert DomainHealth record with results
  //    - Compare with previous DomainHealth record:
  //      * NEW blacklist hits (not in previous blacklistHits) → notifyBlacklistHit (one-time alert)
  //      * REMOVED blacklist hits → notifyBlacklistDelisted
  //      * DNS failures where previous was passing → notifyDnsFailure(persistent: false)
  //      * DNS failures where lastDnsCheck was >48h ago and still failing → notifyDnsFailure(persistent: true)
  // 4. Return summary JSON
}

- Blacklist dedup: alert once per NEW listing, suppress repeats
- DNS escalation: first failure = warning, persistent >48h = critical
- Admin only notifications
- Scheduled daily at 8am UTC via cron-job.org (user configures externally)
