1. Add DomainHealth model to prisma/schema.prisma:
```
model DomainHealth {
  id        String   @id @default(cuid())
  domain    String   @unique  // e.g. "outsignal.ai"

  // SPF validation
  spfStatus   String?   // "pass" | "fail" | "missing"
  spfRecord   String?   // Raw SPF record text

  // DKIM validation
  dkimStatus    String?   // "pass" | "partial" | "fail" | "missing"
  dkimSelectors String?   // JSON array of passing selectors, e.g. '["google","default"]'

  // DMARC validation
  dmarcStatus String?   // "pass" | "fail" | "missing"
  dmarcPolicy String?   // "none" | "quarantine" | "reject"
  dmarcRecord String?   // Raw DMARC record text

  // Blacklist status (populated by Plan 03)
  blacklistHits     String?   // JSON array of DNSBL names where listed
  blacklistSeverity String?   // "none" | "warning" | "critical"
  lastBlacklistCheck DateTime?

  // Overall health
  overallHealth String   @default("unknown") // "healthy" | "warning" | "critical" | "unknown"

  // Timestamps
  lastDnsCheck DateTime?
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  @@index([overallHealth])
  @@index([lastDnsCheck])
}
```

<task type="auto">
  <name>Task 2: Build DNS validation library</name>
  <files>src/lib/domain-health/dns.ts</files>
  <action>
Create src/lib/domain-health/dns.ts with pure DNS validation functions using Node.js `dns/promises` (zero external deps):

1. `checkSpf(domain: string): Promise<SpfResult>` — Look up TXT records for `domain`, find the one starting with `v=spf1`, return pass if found, fail if malformed, missing if absent. Wrap in try/catch for ENOTFOUND/ENODATA.

2. `checkDkim(domain: string): Promise<DkimResult>` — For each selector in DKIM_SELECTORS, look up TXT record at `{selector}._domainkey.{domain}`. Return "pass" if at least one found, "partial" if some found, "missing" if none. Collect passing selector names. Use Promise.allSettled for parallel checks.

3. `checkDmarc(domain: string): Promise<DmarcResult>` — Look up TXT record at `_dmarc.{domain}`, parse `p=` directive for policy. Return pass + policy if found, missing if absent.

4. `checkAllDns(domain: string): Promise<DnsCheckResult>` — Run all three in parallel via Promise.all, return combined result.

5. `computeOverallHealth(dns: DnsCheckResult, blacklistHits: string[]): string` — Returns "healthy" if all DNS pass and no blacklist hits. Returns "critical" if any blacklist hits or SPF/DMARC fail. Returns "warning" if DKIM partial or DMARC policy is "none". Otherwise "unknown".

All functions should be resilient to DNS timeouts (set 5s timeout via Resolver options). Log errors with `[domain-health]` prefix.
