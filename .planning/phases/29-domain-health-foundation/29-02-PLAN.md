1. Add BounceSnapshot model to prisma/schema.prisma:
```
model BounceSnapshot {
  id        String   @id @default(cuid())

  // Identity
  workspaceSlug   String
  senderEmail     String   // e.g. "john@outsignal.ai"
  senderDomain    String   // e.g. "outsignal.ai" — derived from senderEmail

  // Cumulative metrics from EmailBison (raw snapshot)
  emailsSent      Int
  bounced         Int
  replied         Int
  opened          Int

  // Daily deltas (computed from previous snapshot)
  deltaSent       Int?     // null on first snapshot
  deltaBounced    Int?
  deltaReplied    Int?

  // Bounce rate (computed: deltaBounced / deltaSent or cumulative)
  bounceRate      Float?   // null if insufficient data (<20 sends)

  // Warmup data (from EmailBison warmup API)
  warmupEnabled   Boolean?
  warmupData      String?  // JSON blob from warmup API response

  // Snapshot date
  snapshotDate    DateTime // Truncated to date (no time), e.g. 2026-03-10T00:00:00Z
  createdAt       DateTime @default(now())

  @@unique([senderEmail, snapshotDate])
  @@index([workspaceSlug, snapshotDate])
  @@index([senderDomain, snapshotDate])
  @@index([snapshotDate])
}
```

2. Run `npx prisma db push` to sync.

3. Create src/lib/domain-health/snapshots.ts:

`captureSnapshots(workspaceSlug: string, apiToken: string): Promise<{ captured: number; errors: string[] }>`
- Create EmailBisonClient with apiToken
- Call getSenderEmails() to get all sender emails with cumulative stats
- For each sender email:
  - Derive senderDomain from email (split at @)
  - Query latest BounceSnapshot for this senderEmail (to compute deltas)
  - Compute deltas: deltaSent = current.emails_sent_count - prev.emailsSent (or null if no prev)
  - Handle counter reset: if current cumulative < prev cumulative, set deltas to null (per CONTEXT.md)
  - Compute bounceRate: if cumulative emailsSent >= 20, use deltaBounced/deltaSent for the day (or cumulative bounced/emailsSent if first snapshot). If <20 total sends, leave null.
  - Upsert BounceSnapshot with @@unique([senderEmail, snapshotDate])
  - snapshotDate = today at midnight UTC

`computeDeltas(senderEmail: string, current: BounceSnapshot, previous: BounceSnapshot | null): { deltaSent: number | null; deltaBounced: number | null; deltaReplied: number | null }`
- Pure function. If no previous, return nulls. If counter reset detected, return nulls. Otherwise subtract.

`computeDomainRollup(domain: string, date: Date): Promise<{ totalSent: number; totalBounced: number; weightedBounceRate: number | null; senderCount: number }>`
- Query all BounceSnapshot rows for domain on given date
- Sum emailsSent and bounced across all senders
- Weighted bounce rate = totalBounced / totalSent (only if totalSent >= 20)
- Return aggregate

`captureAllWorkspaces(): Promise<{ workspaces: number; senders: number; errors: string[] }>`
- Load all workspaces from DB (prisma.workspace.findMany where apiToken is not null)
- For each workspace, call captureSnapshots
- Aggregate results

Apply the 20-send minimum gate per CONTEXT.md decisions: below 20 campaign sends, bounceRate stays null.
