1. **Create `src/app/api/analytics/campaigns/route.ts`** — GET endpoint:
   - Query params: `workspace` (optional, filters to one workspace), `period` ("24h" | "7d" | "30d" | "all", default "30d"), `sort` (column name, default "replyRate"), `order` ("asc" | "desc", default "desc")
   - Compute date range from period: 24h = today, 7d = last 7 days, 30d = last 30 days, all = no date filter
   - Query CachedMetrics: `WHERE metricType = "campaign_snapshot"` + workspace filter + date filter
   - **Aggregate across date range**: For each unique metricKey (campaign), take the LATEST snapshot row (most recent date). This gives current cumulative stats since EB campaign stats are cumulative, not daily deltas. Do NOT sum across days — that would double-count.
   - Parse each row's `data` JSON into CampaignSnapshot
   - Filter out campaigns with < 10 total sends (emailsSent + linkedinConnectionsSent + linkedinMessagesSent) — these lack statistical significance
   - Map to response shape: `{ id: metricKey, name: campaignName, workspace, channels, sent: emailsSent, replyRate, openRate, bounceRate, interestedRate, replied, opened, bounced, interested, copyStrategy, status }`
   - Sort by requested column, in requested order
   - Return JSON: `{ campaigns: [...], total: number, period, filters: { workspace, sort, order } }`

2. **Create `src/app/api/analytics/campaigns/[id]/steps/route.ts`** — GET endpoint:
   - Path param: campaign ID (Outsignal cuid)
   - Query CachedMetrics for this campaign's latest snapshot (most recent date)
   - Parse data JSON, extract `stepStats` array
   - If stepStats is empty/null, also query Reply table directly: GROUP BY sequenceStep WHERE campaignId = id, COUNT replies, COUNT interested (intent IN ('interested', 'meeting_booked')), COUNT objections (intent = 'objection')
   - Also query local Campaign record to get emailSequence JSON — extract step subject lines for display labels
   - Also include LinkedIn steps from Campaign.linkedinSequence if present
   - Return JSON: `{ campaignId, campaignName, steps: [{ step: number, channel: "email" | "linkedin", label: string, sent: number, replied: number, replyRate: number, interestedCount: number, objectionCount: number, intentDistribution: { interested: n, objection: n, not_now: n, ... } }] }`
   - For intent distribution per step: query Reply table grouped by sequenceStep AND intent WHERE campaignId = id
   - Guard division by zero on replyRate (sent > 0 check)
  </action>
