1. **Evolve CachedMetrics** in `prisma/schema.prisma`:

   - Change unique constraint from `@@unique([workspace, metricType])` to `@@unique([workspace, metricType, metricKey, date])`

   - Run `npx prisma db push` to apply (project convention — no migrate dev)

2. **Create `src/lib/analytics/snapshot.ts`** — the core logic:
   - Export `snapshotWorkspaceCampaigns(workspaceSlug: string): Promise<{ campaignsProcessed: number; errors: string[] }>`
   - Steps inside:
     a. Look up workspace by slug to get apiToken
     b. Create EmailBisonClient with workspace apiToken
     c. Call `getCampaigns()` to get all EB campaigns with their stats
     d. Load all local Campaign records for this workspace (to bridge EB integer IDs to Outsignal cuids via `emailBisonCampaignId`)
     e. For each local campaign that has an EB match:
        - Email metrics: use the EB Campaign object fields directly (emails_sent, opened, unique_opens, replied, unique_replies, bounced, interested, total_leads, total_leads_contacted)
        - LinkedIn metrics: COUNT LinkedInAction records grouped by actionType WHERE workspaceSlug matches AND campaignName matches AND status = 'complete'. Get connectionsSent (actionType='connect'), messagesSent (actionType='message'), profileViews (actionType='profile_view'). For connectionsAccepted: COUNT LinkedInAction WHERE actionType='connect' AND result contains '"accepted"'
        - Reply classification stats: COUNT Reply records WHERE campaignId matches, group by intent. Compute interestedReplies (intent IN ('interested', 'meeting_booked')), objectionReplies (intent = 'objection'), classifiedReplies (intent IS NOT NULL)
        - Per-step stats: GROUP Reply records by sequenceStep WHERE campaignId matches. For each step, count replied + interested + objection
        - Compute rates: replyRate, openRate, bounceRate, interestedRate — all guarded against division by zero (totalSent > 0 check). totalSent = emailsSent + linkedinConnectionsSent + linkedinMessagesSent for multi-channel campaigns
        - Channels: parse Campaign.channels JSON to determine ["email"], ["linkedin"], or ["email", "linkedin"]
     f. Upsert into CachedMetrics: workspace=workspaceSlug, metricType="campaign_snapshot", metricKey=campaign.id (Outsignal cuid), date=today YYYY-MM-DD UTC, data=JSON.stringify(snapshot)
   - Use the CampaignSnapshot interface from RESEARCH.md as the data shape (emailsSent, opened, uniqueOpens, replied, uniqueReplies, bounced, interested, totalLeads, totalLeadsContacted, linkedinConnectionsSent, linkedinConnectionsAccepted, linkedinMessagesSent, linkedinProfileViews, classifiedReplies, interestedReplies, objectionReplies, stepStats, replyRate, openRate, bounceRate, interestedRate, campaignName, channels, copyStrategy, status)
   - For campaigns without EB match (LinkedIn-only): still snapshot from local LinkedInAction data

3. **Create `src/lib/analytics/strategy-detect.ts`**:
   - Export `detectCopyStrategy(emailBody: string): Promise<"creative-ideas" | "pvp" | "one-liner" | "custom">`
   - Use `@ai-sdk/anthropic` with `generateText` (NOT generateObject — simple string output)
   - Model: claude-haiku-4-5-20250315 (cheapest, sufficient for classification)
   - Prompt from RESEARCH.md: classify into creative-ideas, pvp, one-liner, or custom
   - Parse response, validate it matches expected values, default to "custom" if unrecognized
   - Export `backfillCopyStrategies(workspaceSlug: string): Promise<number>` — finds all Campaign records with null copyStrategy that have emailSequence, extracts first step body text, runs detectCopyStrategy, updates Campaign.copyStrategy. Returns count of updated campaigns.
