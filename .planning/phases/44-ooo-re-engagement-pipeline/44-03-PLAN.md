  <action>
1. Create `src/app/api/ooo/route.ts` with GET handler:
   - Require admin auth via `requireAdminAuth()`
   - Query params: `workspaceSlug` (optional, default "all"), `status` (optional, default "all")
   - Query `prisma.oooReengagement.findMany()` with:
     - Where: workspaceSlug filter (if not "all"), status filter (if not "all")
     - OrderBy: oooUntil asc (soonest returns first)
     - Include related data: look up Person by personEmail for name
   - Also compute summary stats:
     - `totalOoo`: count where status="pending"
     - `returningThisWeek`: count where status="pending" AND oooUntil between now and now+7days
     - `reengaged`: count where status="sent"
     - `failed`: count where status="failed"
   - Return `{ records: [...], summary: { totalOoo, returningThisWeek, reengaged, failed } }`

2. Create `src/app/api/ooo/[id]/route.ts` with PATCH and DELETE handlers:

   PATCH (reschedule):
   - Require admin auth
   - Body: `{ oooUntil: string }` (new ISO date)
   - Load OooReengagement by id — if status !== "pending", return 400 "Can only reschedule pending records"
   - If triggerRunId exists, call `runs.reschedule(triggerRunId, { delay: newSendDate })` (newSendDate = oooUntil + 1 day)
   - Update record: oooUntil=new Date(body.oooUntil)
   - Return 200 with updated record

   DELETE (cancel):
   - Require admin auth
   - Load OooReengagement by id — if status !== "pending", return 400 "Can only cancel pending records"
   - If triggerRunId exists, call `runs.cancel(triggerRunId)`
   - Update record: status="cancelled", cancelledAt=new Date()
   - Return 200 with updated record

   Both PATCH and DELETE: wrap Trigger.dev SDK calls in try/catch — if the run is already completed/cancelled, log warning but still update local record.
  </action>
