Replace the stub in `trigger/ooo-reengage.ts` with full implementation. The task fires when the delayed task triggers (day after lead's return date).

Task flow:
1. Load the OooReengagement record by `reengagementId` — if status is not "pending", return early (already processed or cancelled)

2. Look up the workspace by slug (need EB base URL + API key from workspace config). Get the EmailBisonClient instance for this workspace.

3. Resolve EB lead ID:
   - If `ebLeadId` is provided in payload, use it
   - Otherwise, look up via EB API: `GET /workspaces/{slug}/leads?email={personEmail}` — extract the lead's EB ID from the response
   - If lead not found in EB, mark status="failed", failureReason="Lead not found in EmailBison", return

4. Find or create Welcome Back campaign:
   - Query local DB: `prisma.campaign.findFirst({ where: { workspaceSlug, name: { contains: "Welcome Back" } } })`
   - If found, use its `emailBisonCampaignId`
   - If not found: look up the original campaign (from `originalCampaignId`), get its `emailSequence` JSON, extract step 2 or 3 body content. Log a warning that no Welcome Back campaign exists — for v1, just attach to the original campaign's EB ID as a fallback (the campaign builder/template creation is complex and better handled as a manual admin task). Set `welcomeBackCampaignId` on the record.
   - IMPORTANT: For this v1 implementation, if no Welcome Back campaign found, try to re-attach to the original campaign. If originalCampaignId resolves to an EB campaign, use that. If neither works, mark failed.

5. Call `ebClient.attachLeadsToCampaign(ebCampaignId, [ebLeadId])`
   - On success: update OooReengagement status="sent", sentAt=new Date(), welcomeBackCampaignId=ebCampaignId

6. Update Person record: clear oooUntil, oooReason, oooDetectedAt (person is no longer OOO)

7. Call `notifyOooReengaged({ workspaceSlug, count: 1, leadEmails: [personEmail] })`

Error handling:
- Wrap EB calls in try/catch — on failure, update status="failed", failureReason=error.message
- Task has retry: { maxAttempts: 2 } — EB transient failures get one retry
- Import PrismaClient at module scope (not inside run — pattern from other tasks)
- Use `emailBisonQueue` for rate limiting (already set in stub)
