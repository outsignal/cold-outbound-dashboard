Replace the stub in `trigger/ooo-reengage.ts` with full implementation. The task fires when the delayed task triggers (day after lead's return date).

Task flow:
1. Load the OooReengagement record by `reengagementId` — if status is not "pending", return early (already processed or cancelled)

2. Look up the workspace by slug (need EB base URL + API key from workspace config). Get the EmailBisonClient instance for this workspace.

3. Resolve EB lead ID:
   - If `ebLeadId` is provided in payload, use it
   - Otherwise, look up via EB API: `GET /workspaces/{slug}/leads?email={personEmail}` — extract the lead's EB ID from the response
   - If lead not found in EB, mark status="failed", failureReason="Lead not found in EmailBison", return

4. Load original campaign copy and adapt via writer agent (per locked decision — Welcome Back content adapted from original campaign steps):
   - If `originalCampaignId` is set, load the campaign: `prisma.campaign.findFirst({ where: { id: originalCampaignId }, select: { emailSequence: true, name: true } })`
   - Parse `emailSequence` JSON → extract step 2 body (fallback to step 3, then step 1)
   - Build reason-based opener: holiday → "Hope you had a great break!", illness → "Hope you're feeling better!", conference → "Hope [eventName] was good!", generic → "Hope all is well!"
   - Call Haiku (`generateText` with `anthropic("claude-haiku-4-5-20251001")`) to adapt the campaign step body into a Welcome Back message:
     - System prompt: "You are a warm, professional email copywriter. Adapt the following campaign email for a re-engagement after an out-of-office period. Keep the original value proposition and call-to-action but add the OOO-aware opener and a thread reference. Tone: warm and casual."
     - User prompt: includes the reason-based opener, the original step body, and instruction to reference the original conversation ("When we last spoke about [topic]...")
   - If no originalCampaignId or campaign not found, fall back to a generic Welcome Back message using just the reason opener + workspace context
   - Store the adapted message body for use in step 5

5. Find or create Welcome Back campaign:
   - Query local DB: `prisma.campaign.findFirst({ where: { workspaceSlug, name: { contains: "Welcome Back" } } })`
   - If found, use its `emailBisonCampaignId`
   - If not found: try to re-attach to the original campaign. Look up CampaignDeploy for originalCampaignId to get emailBisonCampaignId. If neither works, mark failed.
   - Set `welcomeBackCampaignId` on the record.

6. Call `ebClient.attachLeadsToCampaign(ebCampaignId, [ebLeadId])`
   - On success: update OooReengagement status="sent", sentAt=new Date(), welcomeBackCampaignId=ebCampaignId

7. Update Person record: clear oooUntil, oooReason, oooDetectedAt (person is no longer OOO)

8. Call `notifyOooReengaged({ workspaceSlug, count: 1, leadEmails: [personEmail] })`

Error handling:
- Wrap EB calls in try/catch — on failure, update status="failed", failureReason=error.message
- Task has retry: { maxAttempts: 2 } — EB transient failures get one retry
- Import PrismaClient at module scope (not inside run — pattern from other tasks)
- Use `emailBisonQueue` for rate limiting (already set in stub)
- Campaign copy adaptation failure is non-blocking — fall back to generic opener if Haiku call fails
