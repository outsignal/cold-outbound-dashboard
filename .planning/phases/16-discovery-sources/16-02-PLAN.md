Create `src/lib/discovery/adapters/aiark-search.ts` implementing `DiscoveryAdapter`.

**IMPORTANT:** This is DISTINCT from `src/lib/enrichment/providers/aiark-person.ts` (person enrichment by LinkedIn URL). This adapter uses the same endpoint but with search/filter parameters for bulk discovery.

**Class: `AiArkSearchAdapter implements DiscoveryAdapter`**
- `readonly name = "aiark"`
- `readonly estimatedCostPerResult = 0.003` (same per-call cost as enrichment)

**Endpoint:** `POST https://api.ai-ark.com/api/developer-portal/v1/people`
**Auth:** `X-TOKEN` header with `AIARK_API_KEY` env var (LOW confidence — same caveat as existing aiark.ts. Add a comment: "LOW CONFIDENCE auth header — if 401/403, check docs.ai-ark.com and update AUTH_HEADER_NAME")
**Timeout:** 15s via AbortController

**Filter mapping from DiscoveryFilter to AI Ark params:**
The exact filter field names are MEDIUM confidence. Implement defensively:
- `filters.jobTitles` → `title` or `job_title` (try both formats — use whichever works based on existing aiark-person.ts pattern)
- `filters.seniority` → `seniority` (string array or string)
- `filters.locations` → `location` (string array)
- `filters.industries` → `industry` (string array)
- `filters.companySizes` → `company_size` or `employee_count` (string array)
- `filters.keywords` → `keywords` or `q` (join with space)
- `filters.companyDomains` → `company_domain` (string array)

Build the request body by including only non-empty filter fields. Include `page` (zero-based int from pageToken, default 0) and `size` (min(limit, 100)).

**Response validation with Zod (.passthrough()):**

Handle both response shapes defensively: data might be at root level as an array OR nested under `data` key. Check `Array.isArray(raw)` first, then fall back to `raw.data`.

Create `src/lib/discovery/adapters/serper.ts` — a multi-method adapter that does NOT implement DiscoveryAdapter (Serper is query-based, not filter-based). Instead, it exports a `serperAdapter` object with three methods.

**Endpoint:** `POST https://google.serper.dev/search`
**Auth:** `X-API-KEY` header with `SERPER_API_KEY` env var
**Timeout:** 10s via AbortController
**Content-Type:** `application/json`

**Shared helper function:**
```typescript
async function serperPost(body: Record<string, unknown>): Promise<unknown>
```
Handles auth, timeout, error checking, JSON parsing. Throws on non-200. Throws with `status: 429` on rate limit.

**Export types:**
```typescript
export interface SerperWebResult {
  title: string;
  link: string;
  snippet: string;
  position: number;
}

export interface SerperMapsResult {
  company: string;         // place title
  address?: string;
  phone?: string;
  website?: string;
  companyDomain?: string;  // extracted from website URL
  rating?: number;
  ratingCount?: number;
  cid?: string;            // Google Maps CID
}
```

**Method 1: `searchWeb(query: string, num?: number): Promise<{ results: SerperWebResult[]; costUsd: number; rawResponse: unknown }>`**
- POST body: `{ q: query, type: "search", num: num ?? 10 }`
- Map `raw.organic` array to SerperWebResult
- `costUsd: 0.001` (1 credit per call)

**Method 2: `searchMaps(query: string): Promise<{ results: SerperMapsResult[]; costUsd: number; rawResponse: unknown }>`**
- POST body: `{ q: query, type: "places" }`
- Map `raw.places` array to SerperMapsResult
- Extract `companyDomain` from `website` URL: `new URL(website).hostname.replace(/^www\./, "")`
- `costUsd: 0.001`

**Method 3: `searchSocial(query: string, platform?: "reddit" | "twitter"): Promise<{ results: SerperWebResult[]; costUsd: number; rawResponse: unknown }>`**
- Prefix query with `site:reddit.com` or `site:twitter.com` (default: reddit)
- POST body: `{ q: "${sitePrefix} ${query}", type: "search", num: 10 }`
- Map `raw.organic` array to SerperWebResult
- `costUsd: 0.001`
- NOTE: Social results are NOT staged to DiscoveredPerson (per CONTEXT.md: "signal data, not contact records"). The tool returns raw results for the agent. Phase 18 handles SignalEvent creation.

**Key decision from CONTEXT.md:** Maps results are company-level records. When staged via the agent tool (in Plan 03), they use `discoverySource: "serper-maps"` to distinguish from web/social. The SerperMapsResult fields map to DiscoveredPersonResult as:
- `company` ← `place.title`
- `phone` ← `place.phone`
- `companyDomain` ← extracted from `place.website`
- `location` ← `place.address`
- `firstName`, `lastName`, `email`, `jobTitle` all undefined (company-level, not person)

Export `serperAdapter` as a const object (not a class — it doesn't implement DiscoveryAdapter since it uses queries, not filters).

const DirectoryPersonSchema = z.object({
  name: z.string().optional(),

**Extraction prompt:**
```
Extract all people/contacts from this directory page. For each person, capture:
- Full name (or first/last name separately)
- Email address
- Job title or role
- Company or organization name
- Phone number
- LinkedIn profile URL

If a name cannot be split into first/last, put the full name in the 'name' field.
Return an array of person objects. Only include entries that have at least a name or email.
```

**Implementation:**
```typescript
export async function extractDirectory(url: string): Promise<DirectoryExtractionResult>
```

1. Create Firecrawl client (same pattern as firecrawl-company.ts: `new Firecrawl({ apiKey: process.env.FIRECRAWL_API_KEY })`)
2. Call `client.extract()` with:
   - `urls: [url]`
   - `prompt: EXTRACTION_PROMPT`
   - `schema: z.object({ people: z.array(DirectoryPersonSchema) }) as any` (the `as any` cast is REQUIRED — Zod version mismatch, same as firecrawl-company.ts)
3. Timeout: `Promise.race` with 45s timeout (directory pages can be large, longer than company extract)
4. Parse result: expect `result.data.people` as array
5. Validate each record using `isValidExtraction()`:
   ```typescript
   function isValidExtraction(record: z.infer<typeof DirectoryPersonSchema>): boolean {
     const hasIdentity = Boolean(record.name || record.firstName || record.email);
     const emailOk = !record.email || /^[^\s@]+@[^\s@]+\.[^\s@]+$/.test(record.email);
     const linkedinOk = !record.linkedinUrl || record.linkedinUrl.includes("linkedin.com");
     return hasIdentity && emailOk && linkedinOk;
   }
   ```
6. Split name into firstName/lastName if `name` is provided but `firstName`/`lastName` are not: split on first space.
7. Map valid records to `DiscoveredPersonResult[]`:
   - `firstName` ← `record.firstName` or first part of `record.name`
   - `lastName` ← `record.lastName` or rest of `record.name`
   - `email` ← `record.email`
   - `jobTitle` ← `record.jobTitle`
   - `company` ← `record.company`
   - `phone` ← `record.phone`
   - `linkedinUrl` ← `record.linkedinUrl`
   - `companyDomain` — if email present, derive domain (skip free email providers). Import or inline a simple free-provider check.

**Error handling:**
- Firecrawl timeout: throw descriptive error
- Empty/null result: return `{ people: [], validCount: 0, skippedCount: 0, costUsd: 0.001, rawResponse: null }`
- Extraction returns non-array: log warning, return empty
